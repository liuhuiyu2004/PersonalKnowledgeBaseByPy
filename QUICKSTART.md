# 快速开始指南

## 🎯 5 分钟快速上手

### 第一步：安装依赖

```bash
install.bat
```

### 第二步：配置（可选）

```bash
copy .env.example .env
```

如需 AI 功能，编辑 `.env` 添加 OpenAI API 密钥：
```ini
openai_api_key=sk-xxxxxxxxxxxxxxx
```

**不配置也可以正常使用**，系统会使用降级方案。

### 第三步：启动服务

```bash
start.bat
```

启动成功提示：
```
🚀 Personal Knowledge Base 启动成功!
📊 API 文档：http://127.0.0.1:8000/docs
🌐 Web 界面：http://127.0.0.1:8000/
```

### 第四步：开始使用

访问：**http://127.0.0.1:8000**

## 📱 主要功能

### 1️⃣ 创建知识

1. 点击"新建知识"
2. 填写标题和内容（支持富文本）
3. 添加标签
4. 点击"保存"

### 2️⃣ 网络采集

**网页抓取：**
1. 输入文章 URL
2. 勾选"自动保存"和"AI 生成摘要"
3. 点击"开始抓取"

**网络搜索：**
1. 选择"网络搜索"标签页
2. 输入关键词
3. 选择搜索引擎（DuckDuckGo 或 Bing）
4. 勾选"自动保存"
5. 点击"开始搜索"

### 3️⃣ 搜索

1. 点击"搜索"
2. 输入关键词
3. 可使用标签筛选

### 4️⃣ 标签管理

1. 点击"标签管理"
2. 创建新标签（可自定义颜色）
3. 编辑或删除已有标签

## 💡 使用技巧

### 标签管理
- **颜色编码**：给不同类别设置不同颜色
- **层次结构**：使用前缀（如 `Python/基础`）

### 内容组织
- **原子化**：每条知识聚焦一个主题
- **结构化**：使用标题、列表等格式
- **关联**：通过标签建立知识联系

## ❓ 常见问题

**Q: 没有 API 密钥能用吗？**  
A: 可以！系统会使用基于规则的降级方案。

**Q: 数据库在哪里？**  
A: 项目根目录的 `knowledge_base.db` 文件。

**Q: 如何修改端口？**  
A: 编辑 `.env` 文件：`port=8080`

**Q: 爬虫失败怎么办？**  
A: 某些网站有反爬机制，建议更换目标网站。
